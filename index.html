How to Reduce BigQuery Costs

Ranked by typical impact. Most teams can reduce BigQuery query costs 50-90% with partitioning and column selection alone.

Query optimization

  • Never SELECT *. SELECT * scans every column. List only the columns you need — a 20-column table querying 3 columns costs ~15% as much.
  • Partition tables by date. A daily-partitioned table with a WHERE date = '2026-05-15' filter only scans that day's partition. 365× cheaper for single-day queries.
  • Add require_partition_filter. Prevents accidental full-table scans: ALTER TABLE t SET OPTIONS (require_partition_filter=true)
  • Use LIMIT for exploration. LIMIT 100 does NOT reduce scan cost — it runs the full query. Use a sample: WHERE RAND() < 0.01.
  • Dry run before executing. Add --dry_run in bq CLI or set dryRun: true in API to see bytes scanned before paying.

Storage optimization

  • Set table expiration. Temporary tables and staging data should expire: ALTER TABLE t SET OPTIONS (expiration_timestamp=TIMESTAMP_ADD(CURRENT_TIMESTAMP(), INTERVAL 7 DAY))
  • Let partitions age to long-term. Partitions untouched for 90 days automatically drop to $0.010/GB. Don't run unnecessary UPDATE queries that reset the clock.
  • Use clustering after partitioning. Clustering by user_id, country, or product_id after partition date reduces scan cost further for filtered queries.
  • Delete unused datasets. Audit datasets regularly. Dev/test tables often accumulate to hundreds of GB over time.
  • Batch load instead of streaming. Batch loads (bq load, GCS → BQ) are free. Streaming inserts cost $0.010/200MB — avoid for analytics ETL.

GA4 BigQuery cost reduction

  • Always filter by _TABLE_SUFFIX. GA4 stores daily tables: events_20260515. Without a date filter, every query scans the full history.
  • Use pseudo-columns. GA4 BigQuery export columns like event_params are ARRAY<STRUCT> — always use UNNEST with a WHERE clause to avoid scanning all nested data.
  • Create aggregate tables. Instead of re-scanning raw events daily, create a materialized daily summary table. Query the summary instead of raw events for dashboards.
  • Sample for exploration. For exploratory queries, add AND RAND() < 0.1 to work on 10% of data at 10% of the cost.

Architecture for cost control

  • Materialized views. Pre-aggregate commonly queried metrics. BigQuery automatically refreshes on source table updates. Queries against the MV scan only the aggregated data.
  • Data Studio / Looker caching. Enable dashboard query caching — repeated identical queries serve from cache without re-scanning.
  • Budget alerts. Set project-level and dataset-level cost alerts in GCP Billing so spikes are caught before end of month.
  • Slot reservations at scale. If you consistently scan 400+ TB/month, Enterprise Edition slots at $2,400/month base may be cheaper than on-demand $2,500/month at 400TB.

BigQuery Cost – FAQ

How is BigQuery pricing calculated?+
BigQuery on-demand pricing charges $6.25 per terabyte (TB) of data scanned by your queries. The first 1 TB per month is free. Storage costs $0.020/GB/month for active data and $0.010/GB/month for long-term storage (tables not modified in 90+ days) — with 10 GB/month free. Streaming inserts cost $0.010 per 200MB. BigQuery does not charge for: batch data loads, queries on views, or data exports to GCS in the same region. Use the Query Cost calculator above to estimate your specific workload.
Does LIMIT reduce BigQuery query cost?+
No. LIMIT does not reduce the amount of data scanned — it only limits the rows returned in the result. BigQuery scans data before filtering, so SELECT * FROM table LIMIT 100 costs the same as SELECT * FROM table. To reduce scan cost: use partitioned tables with date filters, SELECT specific columns instead of *, use table sampling (TABLESAMPLE SYSTEM(1 PERCENT)), or add a WHERE clause that filters on a partitioned or clustered column.
How much does GA4 BigQuery export cost?+
The export itself is free. You pay for: storage of the exported data ($0.020/GB/month active) and the cost of queries you run against it. A medium site with 50K daily sessions generates roughly 2-5 MB of event data per day — about 60-150 MB/month. Annual storage: ~1-2 GB = $0.02-$0.04/month. Query costs depend on how often you query and whether you use partition filters (_TABLE_SUFFIX). Without partition filters, a full-history scan of 12 months of data for a high-traffic site can cost $1-5 per query. With _TABLE_SUFFIX date filters: fractions of a cent. The GA4 Cost Calculator tab above models this precisely for your traffic level.
When should I switch to flat-rate (slot) pricing?+
Flat-rate makes economic sense when your monthly on-demand spend consistently exceeds ~$3,000-$4,000/month ($6.25/TB × approximately 480-640 TB/month). Enterprise Edition starts at 100 slots for $2,400/month. However, slots also guarantee performance — if you have latency-sensitive dashboards or competitive SLAs, slots guarantee a fixed compute capacity regardless of concurrent query load. Use the Slots calculator tab to compare your current on-demand spend against flat-rate options and find your break-even point.
What is the difference between active and long-term BigQuery storage?+
Active storage costs $0.020/GB/month — applies to any table or table partition that has been modified in the last 90 days. Long-term storage costs $0.010/GB/month (50% cheaper) — automatically applies to tables or partitions that haven't been modified for 90 consecutive days. For partitioned tables, each partition ages independently. A 2-year-old partition of a daily-partitioned table will be long-term even if you insert new data into today's partition. This means large historical archives naturally drop to the lower price without any manual action.
